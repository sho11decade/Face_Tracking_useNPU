# ロードマップ

この文書は「今すぐ着手すべき実装」と「その先の目標」を分けて整理するためのものです。`TECHNICAL_DETAILS.md` の全体方針を踏まえつつ、現時点のコードベースとの差分が分かるようにしています。

## 現在地

実装済みの基盤:

- CMake による分割ビルド構成
- 開発用アプリ `tracker_dev`
- Media Foundation によるカメラ列挙と単発フレーム取得
- 推論/追跡/マッピングの最小データ経路

まだ骨格段階の領域:

- 実推論
- フィルタリング
- 描画/UI
- 設定ロード
- 実時間パイプライン化

## 近接タスク

次の数ステップで価値が高いものを優先しています。

1. キャプチャを単発取得から連続取得へ拡張する
2. `configs/default.json` を実際の初期化に接続する
3. `InferenceService` に実モデル読み込みの最小経路を追加する
4. `TrackingService` に平滑化や欠損時処理の土台を入れる
5. `tracker_dev` でフレーム更新ループと基本ログを持たせる

近接タスクの完了条件:

- 手動確認だけでなく、最低限の回帰確認ができる
- カメラ入力から推論入力までの経路がコード上で連続する
- スタブと実処理の境界が README と docs で説明できる

## 中期目標

1. OpenVINO を使った顔検出の実装
2. 顔ランドマークまたは頭部姿勢推定の実装
3. `NPU -> GPU -> CPU` の実デバイスフォールバック整備
4. OneEuro Filter などの時系列安定化
5. DirectX11 プレビューとデバッグオーバーレイの初期実装
6. Dear ImGui による設定/性能表示パネルの接続

この段階で、設計書上の主パイプラインが実コードとして見え始める想定です。

## 長期目標

3. キャッシュや初回コンパイル時間を含む OpenVINO 運用改善

## 進め方の方針

- まずは「計測できる最小パイプライン」を作る
- 実装済み経路を保ちつつ、各段をスタブから実処理へ置き換える
- 大きな抽象化よりも、デバッグしやすい直線的な構成を優先する
